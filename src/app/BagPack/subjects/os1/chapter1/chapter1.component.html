Operating System Overview

An Operating System (OS) is an interface between a user and computer hardware

Functions of Operating System

1) User Interface =

  • User interface is a paltform for users and other applications running on the Operating system to communicate with each other

    • 2) Resource Management =

  • Devices connected to a computer system , such as memory, storage, I/O devices etc. re called as resources of the computer
  • Operating System controls these resources and enable efficient use of available resources

    • 3) Task Management =

  • Task Management is the part of opearting system that control the running of one or more programs (tasks) whithin the computer at the same time

    • 4) File Management =

  • File management refers to the process of organizing , storing and manipulating files on a computer system.

    • 5) Utilities =

  • A utility is a small program that provides an addition to the capabilities provided by operating system.

    • Operating System Operations

    1) Dual mode operation

  • User Mode
  • Kernel Mode : (Also called as supervisor, system or privileged mode)

    • 2) Timer

  • We can use the timer to prevent a user program from running too long

    • Operating System Structure


    1) Simple Structure

    MS-DOS Structure



    Advantages =>

  • MS-DOS return to provide the most functionality in the list space

    • Disadvantages =>

  • Not Divided into Module
  • Even if MS-DOS has some structure , it's interfaces and levels of functionality are not well separated

    • 2) Layered Approach




    Advantages =>

  • Simple to design and implement
  • Easy to modify and anhance
  • Easy to debug
  • Data and operations of layer is hidden from other layers

    • Disadvantages =>

  • It needs careful definition of layer interface
  • Number of layers must be moderate
  • As higher-level layer call lower layer, Unnecessary data needs to pass through various layer . Each layer adds overhead to the system call

    • 3) Micro-Kernels

  • In this OS structured by removing all nonessential components from kernel and implemented as system and user level programs

    • Advantages =>

  • Easy to extend
  • It is portable
  • More secure and reliable
  • All new services can be added to user space without modifying the kernel

    • Disadvantages =>

  • The performance of microkernel decreases due to increased function overhead

    • 4) Modules

  • Modules are similar to layers in that each subsystem has clearly defined tasks and interfaces, but any module is free to contact any other module, eliminating the problems of going through multiple intermediary layers (Like layered Approach).
  • The kernel is relatively small in this architecture, similar to microkernels, but the kernel does not have to implement message passing since modules are free to contact each other directly. Eg: Solaris, Linux and MacOSX

    • Booting

  • The procedure of starting a computer by loading the kernel is known as booting the system
  • To make operating system available , it should make availbale by hardware . For this purpose , hardware should know where the kernel is or how to load kernel (OS).
  • Small program known as bootstrap program or bootstrap loader loctes the kernel , loads it into main memory and start execution

    • System Calls

  • A system call is a way for programs to interact with the operating system
  • There are 5 different categories of system calls
    1. Process Control
    2. File Management
    3. Device Management
    4. Information maintenance
    5. Communication

    1] Process Control

    • These system calls deal with processes such as process creation, process termination etc
    • getpid() , getppid() , fork() etc.

    2] File related system calls

    • These system calls are responsible for file manipulation such as creating a file, reading a file, writing into a file etc.

    3] Device Management

    • These system calls are responsible for device manipulation such as reading from device , writing into device , accesing I/O devices etc.

    4] Information maintenance

    • These system calls handle information and its transfer between the operating system and the user program.
    • settimer(), sleep(), etc.

    5] Communication

    • These system calls are useful for interprocess communication. They also deal with creating and deleting a communication connection.
    • pipe(), mmap(), etc.
